**用户学习反馈类**

**1. `/analytics/user/overview`**  
接口用途：查看用户在做题上的整体总览，包括提交总数、AC 数、AC 率、平均耗时、平均内存。  
可以看出什么：
- 用户整体练习量是否足够
- 用户当前通过率高不高
- 用户代码在运行效率上是偏稳定还是偏波动
- 用户当前总体学习状态是良好、一般还是偏弱  
意义：该接口用于从整体层面刻画用户的做题情况，帮助用户快速了解自己的综合学习表现，也为后续查看趋势、语言偏好和错误结构提供基础参照。

**2. `/analytics/user/daily?days={N}&userId={id}`**  
接口用途：查看用户最近 N 天每天的提交总数、AC 数、AC 率、平均耗时、平均内存。  
可以看出什么：
- 用户最近一段时间是否保持持续练习
- 用户的通过情况是在提升还是下降
- 用户是否存在某几天集中练习、某几天完全中断的情况
- 用户最近的代码效率是否在改善  
意义：该接口用于描述用户的时间趋势变化，帮助用户观察自己最近的学习节奏和训练稳定性，也能反映短期内学习效果是否在改善。

**3. `/analytics/user/language?userId={id}`**  
接口用途：查看用户在不同编程语言上的提交总数、AC 数和 AC 率。  
可以看出什么：
- 用户最常使用哪种语言
- 用户在哪种语言上的通过率更高
- 用户是否存在“常用一种语言但效果不好”的情况
- 用户是否已经形成相对稳定的语言偏好  
意义：该接口用于分析用户的编程语言使用结构，帮助用户了解自己在哪种语言上更熟练、更稳定，从而辅助其优化做题时的语言选择。

**4. `/analytics/user/verdict?userId={id}`**  
接口用途：查看用户的 verdict 分布，如 AC、WA、RE、CE、TLE 各自的数量。  
可以看出什么：
- 用户最常见的失败类型是什么
- 用户更容易出现逻辑错误、运行错误还是超时问题
- 用户当前做题瓶颈是算法思路问题还是实现细节问题
- 用户整体错误结构是否健康  
意义：该接口用于刻画用户错误类型的组成结构，帮助用户明确自己经常错在什么地方，从而更有针对性地改进解题思路、编码习惯和调试方法。

**5. `/analytics/user/language/daily?days={N}&userId={id}`**  
接口用途：查看用户最近 N 天在不同语言上的使用趋势。  
可以看出什么：
- 用户最近是否在切换编程语言
- 某种语言的使用频率是上升还是下降
- 用户是否开始尝试新的语言解决问题
- 用户语言偏好是否发生变化  
意义：该接口用于反映用户语言使用习惯的时间变化，帮助用户观察自己在学习过程中是否形成了新的语言偏好，以及不同语言是否正在逐渐成为主要解题工具。

**6. `/analytics/user/verdict/daily?days={N}&userId={id}`**  
接口用途：查看用户最近 N 天 verdict 的变化趋势。  
可以看出什么：
- 最近的 AC 数有没有上升
- WA、RE、TLE 等错误类型是否有减少
- 用户近期的做题质量是否在改善
- 用户是否出现了某类错误持续偏高的情况  
意义：该接口用于描述用户判题结果结构的时间变化，帮助用户判断自己最近的练习是否有效，以及某些错误类型是否正在被逐步克服。

**7. `/analytics/user/problems?limit={N}&userId={id}`**  
接口用途：查看用户在不同题目上的提交情况，包括提交次数、AC 数、AC 率、平均耗时、平均内存。  
可以看出什么：
- 用户在哪些题目上反复尝试
- 哪些题目已经掌握，哪些题目仍然困难
- 用户是否存在“长期卡在少数题目上”的现象
- 某些题目上是否已经从做错变成做对  
意义：该接口用于按题目维度分析用户的完成情况，帮助用户识别自己已经掌握的题目与仍需加强的题目，从而形成更具体的学习反馈。

**8. `/analytics/user/recent?limit={N}&userId={id}`**  
接口用途：查看用户最近若干次提交的结果明细。  
可以看出什么：
- 用户最近集中在哪些题目上练习
- 最近使用了什么语言
- 最近是做对还是做错
- 最近的失败主要集中在哪些题目或哪类语言环境下  
意义：该接口用于提供用户最近行为的即时反馈，帮助用户快速回顾近期练习情况，定位最近最需要关注的问题题目和错误提交。

**平台运行监控类**

**9. `/analytics/summary?date={YYYY-MM-DD}`**  
接口用途：根据日期查询全站总提交数、总 AC 数、平均耗时、平均内存开销。  
可以看出什么：
- 某一天平台整体提交规模有多大
- 某一天平台总体通过情况如何
- 某一天提交程序的平均运行效率如何
- 该接口是否适合作为日报和管理统计的基础数据源  
意义：该接口用于提供最基础的全站日级统计数据，为平台日报、数据报表和管理员总览页面提供底层支撑，是平台运行监控的基础入口。

**10. `/analytics/admin/overview?date={YYYY-MM-DD}`**  
接口用途：查看某一天平台总览，包括提交数、AC 数、AC 率、平均耗时、平均内存和活跃用户数。  
可以看出什么：
- 平台当天是否活跃
- 平台总体成功率高不高
- 判题结果是否稳定
- 活跃用户数多不多  
意义：该接口用于从管理视角快速概括平台在某一天的运行状态，是管理员首页总览卡片的核心数据来源，便于快速感知平台当天是否运行正常。

**11. `/analytics/admin/daily?days={N}`**  
接口用途：查看最近 N 天平台整体趋势，包括每日提交数、AC 数、AC 率、平均耗时、平均内存。  
可以看出什么：
- 平台热度是在上升、稳定还是下降
- AC 率是否长期稳定
- 平台负载是否有明显波动
- 最近是否出现异常时段或异常日期  
意义：该接口用于观察平台整体运行趋势，帮助管理员从时间维度判断平台是否持续活跃、运行是否平稳，并为长期容量规划提供依据。

**12. `/analytics/admin/hourly?date={YYYY-MM-DD}`**  
接口用途：查看某一天 24 小时内每小时的提交数、AC 数、AC 率和活跃用户数。  
可以看出什么：
- 平台高峰主要集中在哪些时段
- 某些小时是否有流量突增
- 活跃用户是否在特定时段集中
- 高峰时段是否伴随通过率下降或资源压力上升  
意义：该接口用于刻画平台在小时级别上的运行情况，帮助管理员识别高峰时段和潜在压力时段，为资源预留、任务调度和容量优化提供参考。

**结构分布与异常诊断类**

**13. `/analytics/admin/language/daily?days={N}`**  
接口用途：查看平台语言趋势。  
可以看出什么：
- 不同语言在一段时间内的使用变化
- 平台语言生态是否发生变化，例如 Python 使用量是否明显上升
- 某种语言的使用是否持续下降
- 某种语言是否突然出现异常波动  
意义：该接口用于观察平台语言使用结构的长期变化，为判题环境维护、镜像资源规划及异常语言环境排查提供依据。

**14. `/analytics/admin/language?date={YYYY-MM-DD}`**  
接口用途：查询某一天的语言分布。  
可以看出什么：
- 某一天用户主要使用哪些语言
- 平台主流提交语言是什么
- 不同语言的使用比例是否存在失衡
- 某种语言是否需要重点维护和优化  
意义：该接口用于反映某一时刻平台的语言使用结构，帮助管理员了解主流编程语言分布，并为语言环境维护和资源侧重点分配提供参考。

**15. `/analytics/admin/verdict/daily?days={N}`**  
接口用途：查看最近的 verdict 趋势。  
可以看出什么：
- AC、WA、RE、CE、TLE 等结果最近怎么变化
- RE 是否持续上升
- CE 是否连续偏高
- TLE 是否在某段时间内集中增多  
意义：该接口用于观察平台判题结果结构的时间变化，帮助管理员识别某类错误是否持续异常增长，从而发现编译环境、运行环境、题目设计或资源压力等方面的问题。

**16. `/analytics/admin/verdict?date={YYYY-MM-DD}`**  
接口用途：查看某天的 verdict 分布。  
可以看出什么：
- 某一天整体成功率和失败结构
- 当天主要失败类型集中在哪一类
- 平台当天运行是否稳定
- 是否适合作为运维排查入口  
意义：该接口用于反映某一天平台判题结果的整体结构，帮助管理员快速判断平台运行是否稳定，以及当天主要失败类型集中在哪一类。

**17. `/analytics/admin/problems/{problemId}/verdict?date={YYYY-MM-DD}`**  
接口用途：查询某道题的 verdict 分布。  
可以看出什么：
- 该题是否 WA 特别多
- 该题是否 TLE 特别多
- 该题是否 RE 特别多
- 该题是否 CE 特别多  
意义：该接口用于分析单道题目的错误结构，帮助管理员判断该题是否存在题面歧义、边界条件描述不足、测试数据设置不合理或时间限制不合适等问题，从而支持题目治理和题库质量优化。

**排行与热点定位类**

**18. `/analytics/problems/daily?date={YYYY-MM-DD}`**  
接口用途：查询某一天每道题的提交总数、AC 数、平均耗时、平均内存开销。  
可以看出什么：
- 哪些题目在当天提交量更高
- 哪些题目 AC 率更高或更低
- 哪些题目的运行耗时和资源开销更突出
- 哪些题目值得重点关注  
意义：该接口用于提供题目层面的基础日统计结果，为题目排行、热点题识别和题库管理提供底层数据支持。

**19. `/analytics/users/daily?date={YYYY-MM-DD}`**  
接口用途：查询某一天每个用户的提交总数、AC 数、平均耗时、平均内存开销。  
可以看出什么：
- 哪些用户当天最活跃
- 哪些用户通过率更高
- 哪些用户提交量异常集中
- 平台活跃用户群体大致分布如何  
意义：该接口用于提供用户层面的基础日统计结果，为用户排行、活跃用户识别和教学统计分析提供支撑。

**20. `/analytics/admin/problems/top?date={YYYY-MM-DD}&limit={N}`**  
接口用途：查询某一天提交量最高的题目排行，并返回通过率、平均耗时、平均内存等指标。  
可以看出什么：
- 哪些题目最受欢迎
- 哪些题目最容易吸引用户提交
- 热门题的通过率和资源开销表现如何
- 哪些题需要优先维护或重点展示  
意义：该接口用于识别平台中的热门题目，帮助管理员快速定位高关注度题目，并为题库维护、题目运营和教学展示提供依据。

**21. `/analytics/admin/users/top?date={YYYY-MM-DD}&limit={N}`**  
接口用途：查询某一天提交量最高的用户排行，并返回通过率。  
可以看出什么：
- 哪些用户最活跃
- 哪些用户练习最频繁
- 活跃用户的通过情况如何
- 平台主要活跃人群有哪些  
意义：该接口用于识别平台中的高活跃用户，帮助管理员了解用户活跃结构，为教学管理、运营展示和用户行为分析提供参考。

**题目质量评估类**

**22. `/analytics/admin/problems/difficulty?date={YYYY-MM-DD}&limit={N}&label={label}`**  
接口用途：查看某一天题目的难度评估结果，包括难度分数、难度标签、通过率、平均耗时、平均尝试次数等。  
可以看出什么：
- 哪些题更难，哪些题更容易
- 题目难度标签分布是否合理
- 某些题目是否存在“通过率低但热度高”的情况
- 哪些题可能需要重新检查题面或测试数据  
意义：该接口用于对题库中的题目进行难度分层和质量评估，帮助管理员直观了解题目难度结构，并辅助题目治理、题库优化和分层教学。

**23. `/analytics/admin/problems/{problemId}/difficulty/history?days={N}`**  
接口用途：查看某道题最近 N 天的难度变化趋势。  
可以看出什么：
- 某道题的难度是否长期稳定
- 该题的通过率是否逐渐提高
- 用户对该题的掌握程度是否在改善
- 题目难度是否因题面、数据或环境变化而波动  
意义：该接口用于跟踪单题难度的时间变化，帮助管理员判断一题的教学适配性和稳定性，也有助于发现题目修改前后是否带来了明显影响。

**预测与决策支持类**

**24. `/analytics/admin/forecast/submissions?forecastDate={YYYY-MM-DD}`**  
接口用途：查看 Spark 生成的未来每日提交量预测结果。  
可以看出什么：
- 接下来几天平台提交量大致会怎样变化
- 是否可能出现提交量明显增长
- 哪些日期可能成为未来高峰
- 当前系统是否需要预留更多资源  
意义：该接口用于为管理员提供未来提交量的预测结果，为容量规划、后续扩容和资源调度提供决策支持。即使系统当前未实现自动扩容，该接口也已经为未来的弹性扩展预留了数据基础。
